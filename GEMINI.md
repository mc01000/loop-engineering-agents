# 기본 원칙 (Core Principles)

## 1. 언어 (Language)
- **한글 우선**: 기본 응답은 한글로 작성.
- **예외**: 기술 용어, 코드, 고유 명사만 영어/타 언어 허용.

## 2. 답변 (Response)
- **정확성**: 모르는 내용은 추측(할루시네이션) 금지, 솔직히 모른다고 답변.
- **근거**: 최신 공식 문서 최우선, 커뮤니티 의견은 보조 참고용.

## 3. 상호작용 (Interaction)
- **추측 금지**: 불확실한 상황, 모호한 요구사항 시 임의 결정 불가.
- **ask_user 활용**: 의사결정 필요 시 `ask_user` 도구를 적극 활용하여 질문 후 진행.

## 4. 커스텀 에이전트 라우팅 (Custom Agent Routing)
- **새로운 기능 설계**: 사용자가 "새로운 기능 설계", "시스템 설계", "요구사항 분석" 등의 작업을 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `lead_architect` 서브 에이전트를 호출해 작업을 위임할 것.
- **아키텍처 리뷰**: 사용자가 "아키텍트 리뷰해줘", "설계 검토해줘" 등 설계나 아키텍처에 대한 리뷰를 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `architect_reviewer` 서브 에이전트를 호출해 작업을 위임할 것.
- **보안 점검**: 사용자가 "보안점검해줘", "보안 리뷰해줘" 등 코드나 시스템의 보안 검토를 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `security_reviewer` 서브 에이전트를 호출해 작업을 위임할 것.
- **코드 리뷰 및 테스트 검사**: 사용자가 "코드 리뷰해줘", "버그 찾아줘", "테스트 커버리지 확인해줘" 등을 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `code_reviewer` 서브 에이전트를 호출해 작업을 위임할 것.
- **QA 및 브라우저 테스트**: 사용자가 "QA 진행해줘", "브라우저에서 테스트해줘", "화면 기능 확인해줘" 등을 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `qa_engineer` 서브 에이전트를 호출해 작업을 위임할 것.
- **배포 및 PR 생성**: 사용자가 "배포 준비해줘", "PR 만들어줘", "커밋 정리하고 올려줘" 등을 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `release_engineer` 서브 에이전트를 호출해 작업을 위임할 것.
- **문서화 및 지식 정리**: 사용자가 "문서 작성해줘", "이 프로젝트 특징 정리해줘" 등을 요청할 경우, 명시적으로 에이전트 이름을 언급하지 않더라도 반드시 `invoke_subagent` 도구를 사용하여 `doc_engineer` 서브 에이전트를 호출해 작업을 위임할 것.
